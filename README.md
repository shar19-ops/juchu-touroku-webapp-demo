# 受注報告登録メモ（デモ版）

受注登録メモ.xlsx（38項目の受注報告登録フォーム）をWeb化したアプリのデモ版。得意先・担当者・都道府県市区町村のマスタ検索、添付書類の必須判定、発行者→確認1→確認2の3段階検印フロー、A4印刷/PDF出力を備える。

> ⚠️ **これはデモ版です。** 得意先・担当者マスタ（`public/data/customers.json` / `staff.json`）はすべて架空のサンプルデータであり、実在する企業・人物とは一切関係ありません。本番版は別途プライベートリポジトリで管理されており、実データはこのリポジトリには含まれていません。

## デモを試す

以下のURLをブラウザで開くだけで操作できます（インストール不要）。

```
https://shar19-ops.github.io/juchu-touroku-webapp-demo/
```

得意先・担当者検索は上記サンプルデータの範囲でのみヒットします（例: 「サンプル」「デモ」「テスト」などで検索してみてください）。

## ローカルで動かす場合

```bash
git clone https://github.com/shar19-ops/juchu-touroku-webapp-demo.git
cd juchu-touroku-webapp-demo
npm install
npm start
```

ブラウザで `http://localhost:5174` を開く。

## データの扱い方

- サーバー側にデータベースは無く、案件データはブラウザの**localStorageに保存**される（＝別のPC・別のブラウザとはデータが共有されない）
- 案件を他の人に引き継ぐ／別PCで開く場合は、アプリ内の「ファイルを開く」「保存」機能でJSONファイルとして書き出し・読み込みを行う
- 検印フローで発行者が「確認者へ提出」した後は、確認者側の画面は特記事項・添付書類の閲覧・出力・押印以外は編集できないようロックされる（管理者パスワードで解除可能。デモ版のパスワードは `demo-unlock-2026` ※本番版とは異なる専用パスワード）

## 印刷/PDF出力

- A4縦1ページに収まるレイアウト（`@page`のCSS margin: 上9mm/右10mm/下9mm/左18mm）
- フッター左下に会社ロゴ（`public/images/footer-logo.jpg`、元は受注登録メモ.xlsxのシートフッターから抽出）を表示
